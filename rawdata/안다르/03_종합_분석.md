## 1. 사실/추정 재분리
- [사실] 자체몰·CAFE24 기반 D2C, 교환/반품 정책 운영, 핏 민감 애슬레저.
- [추정] 핏 반품 비중 높음, 폐기·이월로 회수 가치 손실, 검수 편차로 데이터 불일치.
- 미확보(데이터 확인 후 확정): 월 반품 건수·사유 분포·폐기 비중·평균 리드타임.

## 2. 문제 구조도 (인과 사슬)
핏 민감 카테고리 → 교환·반품 반복 → 검수·재입고 지연 → 정상 재고 복원 지연 → 시즌 판매 기회손실 + 폐기/이월 → 회수 가치 손실 + CS 분쟁.

## 3. 긴장 지점
- **재고가 묶인다**: 반품이 적체되는 동안 팔 수 있는 재고가 창고에 잠긴다.
- **버려진다**: 폐기·이월로 회수할 수 있던 가치가 사라진다.

## 4. 의사결정자 맵 (운영지표 → KPI 언어)
- 물류/운영: 처리 리드타임 → **48시간 재고복원 SLA**
- 재고기획/MD: 가용 재고 → **즉시 재고 복원 / 시즌 회전**
- CS: 검수 일관성 → **99.99% 정확도, 분쟁 감소**
- 재무: 폐기 손실 → **리퍼·재판매 회수 / 건당 정산 예측성**

## 5. 1순위 파일럿 확정
- 레깅스 단일 카테고리 반품을 48시간 SLA로 시범 운영 → 회수율·리드타임 검증 후 전 카테고리 확장.
